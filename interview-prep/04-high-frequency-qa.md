## Java 基础

名词提醒：

- `synchronized`：Java 内置锁。
- `volatile`：保证可见性但不保证复合操作原子性的关键字。

### 1. `ArrayList` 和 `LinkedList` 的区别

- `ArrayList` 底层是动态数组，随机访问快，尾部追加快，中间插入删除慢
- `LinkedList` 底层是链表，随机访问慢，插入删除如果已经定位到节点会更方便
- 实际业务里大多数列表场景优先用 `ArrayList`

### 2. `HashMap` 为什么线程不安全

- 多线程并发 `put` 时可能出现数据覆盖、链表或树结构不一致
- 扩容时也可能出现竞态问题
- 并发场景应该用 `ConcurrentHashMap`

### 3. `synchronized` 和 `volatile` 的区别

- `volatile` 解决可见性和有序性，不保证原子性
- `synchronized` 同时保证可见性和原子性，还能做互斥
- 一个适合状态通知，一个适合临界区保护

### 4. 为什么线程池不能乱用默认参数

- 默认队列、核心线程数和最大线程数未必适合你的业务流量
- 队列过大可能堆积请求，过小可能频繁拒绝
- 线程池参数要根据任务类型、机器资源、响应时间目标来定

## MySQL

名词提醒：

- `B+ 树`：MySQL 常见索引结构。
- `MVCC`：多版本并发控制。

### 8. 为什么索引快

- MySQL 常用 B+ 树索引，树高低、磁盘 IO 次数少
- 非叶子节点只存键和指针，单页能放更多索引项
- 叶子节点有序，适合范围查询和排序

### 9. 什么是回表

- 通过二级索引查到主键后，再回主键索引或聚簇索引拿整行数据
- 如果查询字段都在索引里，就能走覆盖索引，避免回表

### 10. 最左前缀原则是什么

- 联合索引从最左列开始连续命中，才能更好利用索引
- 跳过中间列会影响后续列的使用

### 11. 事务四大特性

- 原子性
- 一致性
- 隔离性
- 持久性

### 12. 隔离级别有哪些

- 读未提交
- 读已提交
- 可重复读
- 串行化

### 13. MVCC 是什么

- 多版本并发控制
- 通过版本链和 Read View，让读写尽量不互相阻塞
- 常用于实现一致性读

## Redis

名词提醒：

- `Redis`：基于内存的高性能键值数据库。
- `key`：Redis 中数据的键名。

### 14. Redis 为什么快

- 基于内存
- 单线程模型避免大量线程切换
- 数据结构高效
- 网络模型轻量

### 15. 缓存穿透、击穿、雪崩分别是什么

- 穿透：请求查不存在的数据，缓存和数据库都被打
- 击穿：热点 key 过期瞬间大量并发打到数据库
- 雪崩：大量 key 同时过期或 Redis 故障，流量全部压到数据库

### 16. 常见解决方案

- 穿透：布隆过滤器、缓存空值
- 击穿：热点 key 互斥重建、永不过期加异步刷新
- 雪崩：过期时间打散、多级缓存、限流降级

### 17. Redis 和数据库如何保证一致性

- 中小项目常用旁路缓存
- 读：先查缓存，未命中查库并回填
- 写：先更新数据库，再删除缓存
- 强一致成本高，通常按业务要求做权衡

## Spring / Spring Boot

名词提醒：

- `IOC`：控制反转。
- `AOP`：面向切面编程。
- `Bean`：交给 Spring 容器管理的对象。
- `DispatcherServlet`：Spring MVC 请求分发入口。

### 18. IOC 是什么

- 把对象创建和依赖管理交给容器
- 降低耦合，方便测试和扩展

### 19. AOP 是什么

- 面向切面编程
- 把日志、事务、鉴权这类横切逻辑从业务代码中抽离

### 20. Spring Boot 自动装配的核心价值

- 根据依赖和配置自动创建 Bean
- 减少样板配置
- 提升开发效率

### 21. 一次 HTTP 请求如何到达 Controller

- 请求先到 Web 容器
- 再到 DispatcherServlet
- 找到对应 HandlerMapping 和 HandlerAdapter
- 执行 Controller 方法
- 返回对象后序列化响应

### 22. Spring 事务失效常见场景

- 方法不是 `public`
- 自调用导致代理失效
- 异常被吃掉不抛出
- 数据源或传播行为配置不当

## 项目结合题

名词提醒：

- `JWT`：无状态身份凭证。
- `Gateway`：统一入口层。
- `MQ`：消息队列。
- `Nacos`：服务注册发现组件。

### 23. 你项目里 JWT 是怎么做的

- 用户登录成功后生成 JWT
- 同时把 token 写入 Redis
- 网关校验 JWT 后再比对 Redis 中的 token
- 这样可以支持退出登录和 token 失效控制

### 24. 你项目里 Redis 用在哪

- 登录限流
- 登录态校验
- 任务列表缓存

### 25. 你项目里为什么要用网关

- 统一入口
- 统一鉴权
- 统一转发
- Swagger 聚合

### 26. 你项目里为什么要用 MQ

- AI 作业耗时长，需要异步处理
- 行为事件统计不应阻塞主流程
- MQ 可以降低服务间直接耦合

### 27. 你项目里为什么要用 Nacos

- 微服务拆分后需要服务注册发现
- 网关和调用方不必写死地址
- 配合配置中心管理更方便

### 28. 你项目里的真实优化点是什么

- 登录限流
- 任务列表缓存
- 基于查询场景补充联合索引
- 行为事件幂等兜底
