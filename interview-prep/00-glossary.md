# 名词速查表

这份表只解释你这轮准备里最常见、最容易卡住的名词。目标不是学术定义，而是面试能用的版本。

## Java / 并发

- `JUC`：`java.util.concurrent` 包的简称，指 Java 并发编程常用工具包。
- `synchronized`：Java 内置锁，作用是保证临界区互斥访问。
- `volatile`：Java 关键字，保证变量修改对其他线程立即可见，但不保证复合操作原子性。
- `线程池`：预先创建并复用线程的一种机制，用来控制并发和减少线程创建开销。

## Spring

- `IOC`：控制反转，指对象创建和依赖管理交给 Spring 容器，而不是在代码里自己 `new`。
- `AOP`：面向切面编程，把日志、事务、鉴权这类横切逻辑从业务代码中抽离。
- `Bean`：交给 Spring 容器管理的对象。
- `自动装配`：Spring Boot 根据依赖和配置自动创建所需 Bean 的机制。
- `Controller`：接收 HTTP 请求并返回响应的那一层。
- `Service`：封装业务逻辑的那一层。
- `Mapper`：负责数据库访问的那一层，在 MyBatis / MyBatis-Plus 中常见。
- `DTO`：Data Transfer Object，数据传输对象，用于接口入参、出参或服务间传递数据。

## MySQL

- `B+ 树`：MySQL 常见索引结构，适合范围查询和排序。
- `回表`：先通过二级索引查到主键，再根据主键回到聚簇索引取整行数据。
- `最左前缀原则`：联合索引从最左边字段开始连续命中时，索引利用率最高。
- `MVCC`：多版本并发控制，通过版本链和 Read View 提高并发读性能。
- `Read View`：MVCC 中决定“当前查询能看到哪些版本”的可见性规则。
- `事务`：一组要么都成功、要么都失败的数据库操作。

## Redis / 缓存

- `Redis`：基于内存的高性能键值数据库，常用于缓存、登录态、计数器等。
- `缓存`：把热点数据临时放在更快的存储中，减少对数据库的直接访问。
- `旁路缓存`：先查缓存，未命中查数据库并回填；写时更新数据库后删除缓存。
- `缓存穿透`：查询根本不存在的数据，导致请求穿过缓存直接打数据库。
- `缓存击穿`：热点 key 失效瞬间，大量请求同时打数据库。
- `缓存雪崩`：大量 key 同时失效或 Redis 故障，数据库压力激增。

## 鉴权 / 网关

- `JWT`：JSON Web Token，一种无状态身份凭证，用来表达“我是谁”。
- `鉴权`：校验当前请求是否有权限访问目标资源。
- `认证`：确认当前用户是谁，比如登录校验用户名密码。
- `Gateway`：网关，系统的统一入口，常负责路由、鉴权、限流等横切逻辑。
- `白名单`：无需登录或鉴权即可访问的路径集合。

## 微服务 / 调用

- `微服务`：把系统按业务领域拆成多个独立服务的架构方式。
- `Nacos`：阿里系注册中心和配置中心组件，在项目里主要用于服务注册发现。
- `注册中心`：统一记录服务地址和实例信息的组件。
- `服务发现`：调用方通过服务名而不是固定地址找到目标服务的过程。
- `OpenFeign / Feign`：声明式 HTTP 调用工具，让服务间调用更像本地接口调用。
- `lb://service-name`：Spring Cloud 中按服务名走负载均衡调用的路由写法。

## 消息队列

- `MQ`：Message Queue，消息队列的缩写，用于异步、解耦、削峰。
- `RocketMQ`：阿里开源的消息队列组件，本项目用它做 AI 作业和行为事件异步处理。
- `生产者`：发送消息的一方。
- `消费者`：接收并处理消息的一方。
- `幂等`：同一个请求或消息重复执行多次，结果仍然一致。

## 数据访问

- `CRUD`：Create、Read、Update、Delete，增删改查的缩写。
- `MyBatis-Plus`：基于 MyBatis 的增强框架，减少常规 CRUD 样板代码。
- `LambdaQueryWrapper`：MyBatis-Plus 的条件构造器，用 Lambda 写查询条件，字段名更安全。
