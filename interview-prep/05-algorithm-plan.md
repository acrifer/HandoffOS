# 高频算法计划

目标不是刷 300 题，而是把校招后端最常见的 `20-30` 题刷到能稳定写出。

## 刷题原则

- 先按题型刷，不要随机刷
- 每题至少过三遍：
  - 第一遍会做
  - 第二遍限时写
  - 第三遍脱离提示复写
- 面试时重点是：
  - 能快速识别题型
  - 能说出时间复杂度
  - 能写出无明显 bug 的代码

## 每题答题模板

面试时按这个顺序说：

1. 先确认输入输出
2. 识别题型
3. 说核心思路
4. 说时间复杂度和空间复杂度
5. 再开始写

示例：

> 这题我先用哈希表存已经遍历过的值，边遍历边判断 `target - nums[i]` 是否已经出现过。如果出现过就直接返回下标。时间复杂度是 `O(n)`，空间复杂度也是 `O(n)`。

## 代码风格要求

- 变量名清楚，不要全是 `a b c`
- 边界条件先处理
- 写完后至少自测一组空值、单元素、重复值数据

## 最低验收线

- 反转链表
- 两数之和
- 有效的括号
- 二叉树层序遍历
- 二分查找
- 全排列
- 爬楼梯

这 7 类题是后端校招里出现频率很高的基础盘，必须拿下。
