# 简历与模拟面试

## 简历写法

项目经历不要写成“功能列表”，要写成“问题、方案、结果”。

## 名词提示

- `Gateway`：统一入口层。
- `JWT`：无状态身份凭证。
- `Nacos`：注册中心和配置中心组件。
- `Redis`：缓存和登录态常用组件。
- `RocketMQ`：消息队列组件。
- `MVCC`：多版本并发控制。

## LifeOS 项目经历模板

项目名：`LifeOS 知识管理微服务系统`

技术栈：

- `Java 17`
- `Spring Boot 3`
- `Spring Cloud Gateway`
- `Nacos`
- `MyBatis-Plus`
- `MySQL`
- `Redis`
- `RocketMQ`
- `Docker Compose`

职责描述可参考下面 4 条：

1. 负责基于 `Spring Cloud Gateway + JWT + Redis` 的统一鉴权链路设计与实现，在网关层完成白名单放行、token 校验、登录态校验和用户请求头透传，支持退出登录和登录态失效控制。
2. 负责笔记 AI 异步作业链路实现，在笔记服务中完成作业落库和 RocketMQ 投递，在 AI 服务中完成消息消费与处理，并通过 Feign 回调更新作业状态，实现摘要、整理和任务提取等异步能力。
3. 负责 Redis 在业务中的落地，包括登录限流、登录态存储和任务列表缓存，采用旁路缓存与写后删缓存策略降低数据库读取压力。
4. 结合真实查询场景对笔记、任务、AI 作业和行为表补充联合索引，围绕用户、状态和时间维度优化列表与筛选查询。

## 项目亮点表达模板

### 亮点 1：统一鉴权

- 问题：多个业务服务都需要登录校验，重复实现成本高
- 方案：在 Gateway 统一做 JWT 解析和 Redis 登录态校验
- 结果：业务服务只关注自身逻辑，支持退出登录和 token 失效控制

### 亮点 2：AI 异步作业

- 问题：AI 处理耗时不稳定，同步调用会拖慢接口响应
- 方案：先落作业表，再通过 RocketMQ 异步处理，AI 服务处理后回调更新状态
- 结果：前端可立即拿到作业状态，主流程响应更稳定

### 亮点 3：Redis 与查询优化

- 问题：任务列表、登录请求等存在重复访问和并发压力
- 方案：任务列表使用缓存，登录使用 Redis 限流，数据库按查询条件补充联合索引
- 结果：降低数据库读压力，同时增强登录安全性

## 复盘标准

每次模拟结束，只检查这 5 件事：

1. 有没有一上来就把项目讲散
2. 有没有技术词很多，但没有说明为什么这样设计
3. 有没有把“我做了什么”讲成“项目有什么功能”
4. 有没有被 Redis、MySQL、MQ 追问时卡住
5. 有没有回答过长、没有结论
