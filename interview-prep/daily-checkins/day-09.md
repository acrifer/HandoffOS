## 知识速记

- Redis 快的原因主要是内存存储、数据结构高效、单线程模型避免大量上下文切换。
- 常见场景：缓存、登录态、限流、分布式锁、计数器。
- 缓存穿透是访问不存在的数据，击穿是热点 key 失效瞬间并发冲库，雪崩是大量 key 同时失效或 Redis 宕机。
- 对应方案：空值缓存/布隆过滤器、热点互斥重建、过期时间打散和限流降级。
- 面试要先说定义，再说解决方案。

## 标准回答

### Redis 为什么快

Redis 首先是基于内存的，访问速度远快于磁盘；其次它的核心数据结构实现效率高；再加上单线程处理命令避免了大量锁竞争和线程切换，所以整体性能很高。

### 缓存三大问题怎么答

先定义，再给方案。穿透是查不存在的数据，击穿是热点 key 失效，雪崩是大量 key 同时出问题。方案分别是空值缓存或布隆过滤器、热点互斥和异步刷新、过期时间打散及限流降级。
